Visão Geral do Sistema

PuntoTouch é uma plataforma SaaS multi-tenant para gestão de horas trabalhadas, folha de pagamento e faturamento de obras/projetos.

3
Tipos de Usuário
Multi
Empresas (Tenants)
JWT

100%
Cloud-Based

Objetivo do Sistema

Problema que resolve: Empresas de construção/serviços com múltiplas obras precisam:

  • Controlar horas trabalhadas por funcionário em cada obra
  • Aprovar apontamentos via WhatsApp (sem login)
  • Calcular folha de pagamento automaticamente
  • Gerar faturas para clientes com impostos por país
  • Gerenciar múltiplas empresas em um único sistema

Fluxo Principal

  1. Admin/RH cadastra funcionários e obras no sistema
  2. Funcionário marca suas horas trabalhadas via app mobile
  3. Encarregado recebe link por WhatsApp e aprova (sem login)
  4. Sistema calcula automaticamente folha de pagamento e faturamento
  5. Admin/RH exporta relatórios finais

Sistema Multi-Tenant

⚠️ Importante: Todos os dados são isolados por tenant_id. Nenhum tenant pode acessar dados de outro.

Como Funciona

Cada empresa cadastrada é um tenant independente. O sistema permite múltiplas empresas operarem na mesma plataforma com dados completamente isolados.

Implementação Técnica

-- Todas as tabelas possuem tenant_id
CREATE TABLE usuarios (
    id INT PRIMARY KEY,
    nome VARCHAR(255),
    tenant_id INT NOT NULL,  ← Isolamento
    ...
    FOREIGN KEY (tenant_id) REFERENCES tenants(id)
);

-- Toda query filtra por tenant_id
SELECT * FROM usuarios
WHERE tenant_id = ? AND ativo = 1;

Tenants Atuais

ID Empresa Slug Status

Fluxo de Login Multi-Tenant

1. Usuário acessa: puntotouch.nextim.io/login
2. Insere: email + senha
3. Backend valida credenciais
4. Backend busca tenant_id do usuário
5. Gera JWT com: { user_id, tenant_id, tipo }
6. Frontend armazena token
7. Todas as requisições incluem: Authorization: Bearer [token]
8. Backend extrai tenant_id do token
9. Todas as queries filtram por tenant_id automaticamente

Sistema de Apontamento de Horas

Conceito

Funcionários marcam suas horas trabalhadas por obra, divididas em:

  • Normal: 8h-17h (padrão)
  • Extra: 17h-22h (multiplicador 1.5x)
  • Noturna: 22h-6h (multiplicador 2.0x)

⚠️ Importante: Desde a última atualização, obras podem desabilitar extras/noturnas individualmente via checkbox permite_hora_extra e permite_hora_noturna.

Fluxo Completo

1. Funcionário seleciona OBRA
2. Sistema carrega semana atual (Segunda a Sábado)
3. Funcionário preenche horas de cada dia
4. Auto-save a cada alteração
5. Ao finalizar semana: clica "Enviar"
6. Sistema valida: todos os 6 dias preenchidos?
7. Se OK: muda status para "enviado"
8. Gera token único de aprovação
9. Envia WhatsApp para encarregado da obra
10. Encarregado clica link → abre página pública
11. Encarregado revisa, assina e aprova
12. Status muda para "aprovado"
13. Dados disponíveis para cálculo de folha

Status do Apontamento

Status Descrição Ação Permitida
rascunho Ainda não enviado Funcionário pode editar
enviado Aguardando aprovação Encarregado pode aprovar/rejeitar
aprovado Aprovado pelo encarregado Bloqueado para edição
rejeitado Recusado pelo encarregado Funcionário precisa corrigir

Prevenção de Duplicatas

Problema: Funcionário pode enviar 2x a mesma semana (se corrigir erro)

Solução: Se funcionário reenviar, sistema:

  1. Invalida tokens antigos de aprovação
  2. Deleta apontamento anterior (se duplicado)
  3. Mantém apenas o mais recente

Sistema de Aprovação via WhatsApp

Funcionamento

Encarregados aprovam apontamentos sem precisar fazer login no sistema. Tudo acontece via link único enviado por WhatsApp.

Fluxo Técnico

1. Funcionário envia apontamento
2. Sistema gera token único: bin2hex(random_bytes(32))
3. Salva em: aprovacao_tokens
   - token (64 chars)
   - apontamento_id
   - encarregado_id
   - expira_em (7 dias)
4. Monta link: puntotouch.nextim.io/approve/{token}
5. Envia WhatsApp via wa.me com mensagem:

   *NUEVO REGISTRO DE HORAS*
   *Empleado:* João Silva
   *Obra:* 1234 - Obra Teste
   *Semana:* 09/03/2026
   *Total:* 42.0 horas

   Haz clic para aprobar:
   https://puntotouch.nextim.io/approve/abc123...

6. Encarregado clica link
7. Abre página pública (sem login)
8. Mostra: dados do funcionário, obra, horas detalhadas
9. Encarregado:
   - Assina com dedo (signature canvas)
   - Marca checkbox de confirmação
   - Clica "Aprobar" ou "Rechazar"
10. Sistema valida token
11. Atualiza status do apontamento
12. Marca token como usado
13. Mostra mensagem de sucesso

Segurança

  • ✅ Token de 64 caracteres (impossível adivinhar)
  • ✅ Expira em 7 dias
  • ✅ Uso único (após aprovar, token é invalidado)
  • ✅ Vinculado a encarregado específico
  • ✅ Assinatura digital obrigatória
  • ✅ Checkbox de confirmação obrigatório

Tabela: aprovacao_tokens

CREATE TABLE aprovacao_tokens (
    id INT PRIMARY KEY,
    token VARCHAR(64) UNIQUE,
    apontamento_id INT,
    encarregado_id INT,
    tenant_id INT,
    usado TINYINT(1) DEFAULT 0,
    usado_em DATETIME NULL,
    expira_em DATETIME,
    criado_em DATETIME
);

Segurança

Autenticação

  • ✅ JWT (JSON Web Token) com expiração de 7 dias
  • ✅ Bcrypt para hash de senhas (cost 12)
  • ✅ Token validado em TODAS as requisições protegidas
  • ✅ Logout limpa token do localStorage

Isolamento Multi-Tenant

  • ✅ tenant_id em TODAS as tabelas
  • ✅ TODAS as queries filtram por tenant_id
  • ✅ Validação no backend: usuário só acessa seu tenant
  • ✅ Impossível acessar dados de outro tenant via API

Proteções Implementadas

Ameaça Proteção

CSRF CORS configurado + JWT stateless
Session Hijacking JWT com expiração + HTTPS only
Brute Force Bcrypt (custo alto) + rate limiting (planejado)
