PuntoTouch

Sistema Multi-Tenant de Gestão de Obras com Monitoramento Real-Time

React 18 + Vite PHP 8.3 RESTful API Firebase Cloud Messaging V1 JWT Multi-Tenant MySQL/MariaDB

Sobre o Sistema

PuntoTouch é uma plataforma completa de gestão de obras com arquitetura multi-tenant, permitindo que múltiplas empresas utilizem o mesmo sistema com isolamento total de dados.

O sistema oferece controle completo de obras, clientes, funcionários, encarregados e administradores, além de módulos financeiros (faturamento, folha de pagamento), apontamento de horas, aprovações via WhatsApp e analytics.

Frontend

  • React 18
  • Vite
  • Tailwind CSS + DaisyUI
  • Recharts
  • Tabler Icons

Backend

  • PHP 8.3
  • MySQL/MariaDB
  • JWT Authentication
  • Firebase Admin SDK
  • Multi-Tenant Isolation

Arquitetura

Multi-Tenant

Todas as queries incluem filtro por tenant_id garantindo isolamento completo entre empresas.

SELECT * FROM tabela WHERE tenant_id = ? AND ativo = 1

Autenticação

JWT com expiração de 7 dias. Header obrigatório: Authorization: Bearer TOKEN

Notificações

Firebase Cloud Messaging API V1 para push notifications em tempo real.

Clientes

Gestão Completa

CRUD completo com busca em tempo real e vínculo com obras.

Cadastro

  • Nome: Nome completo (obrigatório)
  • Email: Endereço de e-mail
  • Telefone: Contato telefônico
  • Endereço: Localização

Funcionalidades

  • Busca em tempo real por nome, email e telefone
  • Edição inline de dados
  • Exclusão lógica (soft delete)
  • Vínculo com múltiplas obras
  • Contador de obras por cliente

Obras

Gestão de Projetos

Sistema completo com calendário, configuração financeira e controle de impostos.

Configuração de Faturamento

  • Hora Normal: Valor base (padrão: 0)
  • Hora Extra: Valor hora extra (padrão: 0)
  • Hora Noturna: Valor hora noturna (padrão: 0)
  • Multiplicador Extra: Fator multiplicador (padrão: 0)
  • Multiplicador Noturna: Fator multiplicador (padrão: 0)
Todos os campos iniciam em 0 para configuração individual por obra.

Impostos

  • IGI/IVA: calculado automaticamente
  • IRP: imposto sobre rendimento
  • Segurança Social: contribuição previdenciária

Controles

  • Switch: Permitir Hora Extra
  • Switch: Permitir Hora Noturna
  • Switch: Obra Ativa

Funcionários

Gestão de Colaboradores

Cadastro completo com biometria, foto e função.

Cadastro

  • Passaporte: Documento de identificação (obrigatório)
  • Nome: Nome completo
  • Email: Endereço de e-mail (opcional)
  • Telefone: Contato (opcional)
  • Função: Cargo (Pedreiro, Eletricista, etc.)
  • Senha: Senha de acesso

Biometria

Sistema de cadastro biométrico para marcação de ponto segura.

Upload de Foto

  • Galeria: selecionar da galeria do dispositivo
  • Câmera: tirar foto diretamente
  • Conversão automática para WebP (200x200px)
  • Correção EXIF automática (evita rotação)
A função do funcionário aparece no card: Empleado - Pedreiro

Encarregados

Aprovação via WhatsApp

Sistema exclusivo de aprovação de apontamentos via WhatsApp com token de segurança.

Fluxo Completo

  1. Funcionário marca ponto no sistema
  2. Encarregado recebe notificação push
  3. Encarregado acessa link único via WhatsApp
  4. Sistema exibe detalhes (obra, funcionário, horas)
  5. Encarregado assina digitalmente
  6. Aprova ou rejeita
  7. Token de segurança valida autenticidade
Encarregados NÃO precisam de passaporte no cadastro.

Segurança

  • Token JWT único por aprovação
  • Validade: 7 dias
  • Registro de IP e timestamp

Apontamento de Horas

Sistema Semanal

Marcação de ponto em grid semanal com validação automática e reenvio.

Campos

  • Obra: Seleção da obra
  • Horas Normais: Quantidade de horas regulares
  • Horas Extras: Se permitido na obra
  • Horas Noturnas: Se permitido na obra

Validação Automática

  • Verifica se obra permite hora extra
  • Verifica se obra permite hora noturna
  • Total não pode exceder 24h/dia
  • Obra deve estar ativa

Funcionalidades

  • Auto-save a cada alteração
  • Reenvio se rejeitado
  • Status: pendente/aprovado/rejeitado

Monitoramento Real-Time

Visualização em Tempo Real

Funcionalidade exclusiva v2.0. Visualiza funcionários que não marcaram ponto.

Status

  • Pendente: Não enviou nenhum apontamento
  • Incompleto: Faltam dias na semana
  • Rejeitado: Apontamento rejeitado

Notificações

  • Individual: Enviar para funcionário específico
  • Broadcast: Notificar TODOS os funcionários
Atualiza em tempo real. Funcionário sai da lista ao marcar ponto.

Aprovações

Fluxo WhatsApp

Aprovação com assinatura digital e token de segurança.

Processo

  1. Funcionário envia apontamento
  2. Sistema dispara notificação push
  3. Encarregado clica → WhatsApp
  4. WhatsApp abre link único
  5. Página exibe detalhes
  6. Encarregado assina no canvas
  7. Clica em Aprovar/Rejeitar
  8. Sistema registra com timestamp e IP

Assinatura Digital

Canvas HTML5 para assinatura manuscrita com mouse ou toque. Convertida em imagem e armazenada.

Segurança

  • Token JWT único
  • Validade: 7 dias
  • Rastreabilidade: IP + timestamp

Faturamento

Geração Automática

Cálculo de faturas com IGI/IVA automático.

Fórmula de Cálculo

Valor Total = (Horas Normais × Valor Hora Normal) + (Horas Extras × Valor Hora Extra × Multiplicador Extra) + (Horas Noturnas × Valor Hora Noturna × Multiplicador Noturna)

IGI/IVA

Imposto calculado automaticamente sobre o valor total e exibido de forma clara.

Geração

  • Por mês
  • Por obra
  • Consolidação de todos os apontamentos aprovados

Folha de Pagamento

Cálculo de Custos

Folha de pagamento por funcionário.

Diferença:
Faturamento: Quanto a empresa COBRA do cliente (receita)
Folha: Quanto a empresa PAGA aos funcionários (custo)

Cálculo

  • Total de horas normais
  • Total de horas extras
  • Total de horas noturnas
  • Valor total a pagar
  • Impostos (IRP, Segurança Social)
  • Valor líquido

Firebase Integration

Firebase Cloud Messaging API V1

Versão mais recente e segura do Firebase.

Configuração

  • Service Account: Arquivo JSON com credenciais
  • Project ID: Identificador do projeto
  • FCM Token: Token único por dispositivo

Backend

Firebase Admin SDK via API V1:

POST https://fcm.googleapis.com/v1/projects/{PROJECT_ID}/messages:send

Arquivo: backend/config/firebase-config.json

App

Service Worker: public/firebase-messaging-sw.js

Fluxo

  1. Login → app solicita permissão
  2. Firebase gera FCM token
  3. App envia token para backend
  4. Backend armazena vinculado ao usuário
  5. Evento ocorre → backend busca tokens
  6. Backend envia payload para Firebase
  7. Firebase entrega nos dispositivos
Para usar API V1, atualizar app mobile para v2.0.

Segurança

Autenticação e Autorização

  • JWT Authentication com expiração 7 dias
  • Multi-tenant isolation via tenant_id
  • Password hashing com bcrypt
  • HTTPS obrigatório em produção

Proteção

  • SQL Injection prevention via PDO prepared statements
  • XSS protection via output sanitization
  • CSRF tokens em formulários
  • Rate limiting em endpoints críticos

Tokens

  • Tokens únicos por aprovação WhatsApp
  • Registro de IP e timestamp em aprovações
  • FCM tokens armazenados com criptografia
