# 📋 Template .gitignore para Projetos React + Capacitor + Android

Use este arquivo em todos os projetos pra evitar subir arquivos desnecessários pro Git.

## 🚫 O que NUNCA deve subir pro Git

### 1. Node.js
- `node_modules/` - Dependências do npm (reinstala com `npm install`)
- `package-lock.json` - Opcional (alguns devs preferem commitar, outros não)
- `npm-debug.log` - Logs de erro

### 2. Build e compilação
- `dist/` - Build do Vite (gerado com `npm run build`)
- `build/` - Build do Android
- `android/app/build/` - Arquivos temporários do Gradle
- `android/.gradle/` - Cache do Gradle
- `.aab` e `.apk` - Pacotes compilados

### 3. JDK e ferramentas externas
- `jdk-*/` - Java Development Kit (cada dev baixa no próprio PC)
- `openjdk.zip` - Arquivo de instalação do JDK
- Qualquer pasta com binários grandes (>50 MB)

### 4. Credenciais e segurança ⚠️
- `.env` - Variáveis de ambiente (senhas, API keys)
- `*.keystore` - Chave de assinatura Android
- `key.properties` - Configurações do keystore
- Qualquer arquivo com senhas ou tokens

### 5. IDEs e editores
- `.vscode/` - Configurações do VS Code
- `.idea/` - Configurações do IntelliJ/Android Studio
- `*.swp`, `*.swo` - Arquivos temporários do Vim

### 6. Sistema operacional
- `.DS_Store` - macOS
- `Thumbs.db` - Windows
- `desktop.ini` - Windows

### 7. Arquivos temporários
- `*.tmp`, `*.bak` - Backups temporários
- `*.log` - Logs
- `nul` - Arquivo vazio do Windows

## ✅ O que DEVE subir pro Git

- ✅ Código fonte (`src/`, `public/`)
- ✅ `package.json` (lista de dependências)
- ✅ Configurações (`vite.config.js`, `capacitor.config.json`, `tailwind.config.js`)
- ✅ Scripts (`.bat`, `.py`, `.sh`)
- ✅ Documentação (`README.md`, `*.md`)
- ✅ Arquivos de assets pequenos (ícones, imagens < 5 MB)
- ✅ `.gitignore` (sempre!)

## 🔧 Como usar

1. Copia o template acima
2. Cria arquivo `.gitignore` na raiz do projeto
3. Cola o conteúdo
4. Ajusta conforme necessário

## ⚠️ Arquivos já commitados

Se você já commitou arquivos grandes por engano:

```bash
# Remove do Git mas mantém no PC
git rm --cached -r node_modules jdk-17.0.2 *.keystore

# Adiciona .gitignore
git add .gitignore

# Commita
git commit -m "Add .gitignore and remove large files"

# Push
git push
```

## 💡 Dicas

- **Keystore**: NUNCA commitar! Se perder, não consegue mais atualizar o app na Play Store
- **node_modules**: Ocupa muito espaço (200-500 MB), sempre no .gitignore
- **JDK**: Cada dev instala sua versão, não precisa no Git
- **Builds**: São gerados na hora, não precisa versionar
