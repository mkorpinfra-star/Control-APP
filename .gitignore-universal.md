## 💡 Como Usar

### 1. Novo projeto
```bash
# Copia o template
cp .gitignore-universal.md .gitignore

# Remove as explicações e mantém só o código
# (ou copia o template completo no final)
```

### 2. Projeto existente
```bash
# Se já commitou arquivos grandes
git rm --cached -r node_modules vendor build dist *.keystore

# Adiciona .gitignore
git add .gitignore

# Commita
git commit -m "Add universal .gitignore"

# Push
git push
```

---

## ⚠️ Regras de Ouro

1. **NUNCA commitar:**
   - Senhas, tokens, API keys
   - Keystores e certificados
   - node_modules, vendor, venv
   - Builds e arquivos compilados
   - Arquivos > 50 MB

2. **SEMPRE commitar:**
   - Código fonte
   - package.json / requirements.txt / composer.json
   - Configurações (exceto .env)
   - README e docs
   - .gitignore

3. **Dica:**
   - Mantenha este template em `~/templates/.gitignore`
   - Copie para todo projeto novo
   - Ajuste conforme necessário
