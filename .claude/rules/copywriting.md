# REGRAS DE COPYWRITING

---

## 🚨 PESQUISA ANTES DE ESCREVER — OBRIGATÓRIA

> ❌ NUNCA escrever copy sem pesquisar concorrentes antes.
> Copy inventada sem base em pesquisa é VIOLAÇÃO grave.

Se tiver acesso à web:
1. Pesquisar o termo principal no Google — apenas orgânicos
2. Analisar os **5 primeiros resultados orgânicos**
3. Mapear headings h1–h3 dos concorrentes
4. Identificar: que dores abordam, que benefícios prometem, que provas usam
5. Produzir versão superior em profundidade, especificidade e clareza

Se não tiver acesso → informar ao usuário e solicitar briefing com dados reais.

### Regra de Auditoria:
✔ Se auditar qualquer página, verificar se as seções/headings refletem pesquisa real
❌ Se encontrar headings genéricos como "Nossos Serviços" ou "Por que nos Escolher" sem conteúdo específico → flagrar como violação

---

## ESTRUTURA DE PÁGINA DE SERVIÇO

1. Proposta de valor (o que é + para quem)
2. Dor do cliente (problema que resolve)
3. Solução (como resolve)
4. Benefícios objetivos (impacto real)
5. Como funciona (processo simplificado)
6. Prova social (depoimentos, cases, números reais)
7. CTA final

---

## REGRA DE VERDADE ABSOLUTA

❌ Nunca usar números sem fonte real fornecida pelo cliente
❌ Nunca usar estatísticas genéricas
❌ Nunca prometer resultado fora do controle do prestador
❌ Nunca usar depoimentos ou provas sociais inventadas
❌ Nunca usar urgência falsa

✔ Se o cliente não forneceu número real → não colocar número nenhum
A pergunta que guia cada frase: "Isso é verdade e verificável?"

---

## 🚨 PREÇOS E VALORES — PROIBIDO

> NUNCA mencionar preços, custos por m², valores em R$, faixas de preço ou tabelas de custo.
> Preços são exclusivos do cliente e variam por projeto — NÃO COLOCAR NA PÁGINA.

❌ "A partir de R$ 45/m²" | "Custo médio de R$ 3.000" | "Investimento de R$ 800 a R$ 1.500"
❌ Seções como "Quanto custa" com valores numéricos
❌ Tabelas de preço ou comparativos de custo com valores

✔ Em vez de preço → Levar para orçamento: "Solicite um orçamento personalizado"
✔ Mencionar que "valores variam conforme o projeto" sem dar números
✔ Focar em BENEFÍCIOS, QUALIDADE e DIFERENCIAIS — não em preço
✔ Se a pesquisa Google mostrar headings como "Preço por m²" → trocar por heading sobre diferenciais ou processo
✔ A seção de "custo" pode existir, mas deve falar sobre FATORES que influenciam o preço, NUNCA o preço em si

---

## 🚨 DADOS E ESTATÍSTICAS — NUNCA INVENTAR

> PROIBIDO criar números, percentuais ou estatísticas que não foram fornecidos pelo CLIENTE.
> Se o cliente não deu os dados, NÃO COLOCAR.

❌ "350+ instalações completas" (inventado)
❌ "14 anos de experiência" (inventado)
❌ "98% de clientes satisfeitos" (inventado)
❌ "A+++ eficiência energética" (inventado)
❌ Qualquer seção de "números/stats" com dados não fornecidos pelo cliente

✔ Se o cliente forneceu dados reais → usar
✔ Se NÃO forneceu → NÃO criar seção de estatísticas
✔ Em vez de números inventados → destacar DIFERENCIAIS VERIFICÁVEIS (certificações, normas, processo)
✔ A pergunta que guia: "O cliente me deu esse número?" — se NÃO → não colocar

## TOM PROIBIDO

❌ superlativos sem prova | linguagem de guru | urgência fabricada | clichês

## TOM OBRIGATÓRIO

✔ profissional, direto, confiante
✔ frases curtas (máx 20 palavras) | parágrafos curtos (máx 3 linhas)
✔ específico: detalhes concretos valem mais que adjetivos

## CAPITALIZAÇÃO — SENTENCE CASE OBRIGATÓRIO

> Todos os títulos (H1–H6) e subtítulos usam **sentence case**.
> **PRIMEIRA LETRA DA PRIMEIRA PALAVRA = MAIÚSCULA**, restante minúscula. Exceção: nomes próprios.

🚨 **ATENÇÃO: tudo minúsculo também é ERRADO!**
> A IA tende a interpretar "sentence case" como "tudo minúsculo". ISSO É ERRADO.

❌ "Economia Financeira" (Title Case) → ERRADO
❌ "economia financeira" (all lowercase) → ERRADO
✔ "Economia financeira" (sentence case) → CORRETO

❌ "perguntas frequentes" → ERRADO | ✔ "Perguntas frequentes" → CORRETO
❌ "outros serviços" → ERRADO | ✔ "Outros serviços" → CORRETO
✔ "Instalações elétricas no Brasil" ← Brasil é nome próprio, fica maiúscula

## COMPRIMENTO MÍNIMO DO H2 — 5 PALAVRAS

> H2 curto e seco (2-3 palavras) parece telegrama. Precisa ser conversacional.

✔ H2 DEVE ter no mínimo **5 palavras** — tom descritivo e envolvente
❌ "Perguntas frequentes" → ✔ "Dúvidas sobre energia solar que você precisa esclarecer"
❌ "Outros serviços" → ✔ "Conheça outros serviços que oferecemos para o seu projeto"
❌ "Tipos de painéis" → ✔ "Conheça os diferentes tipos de painéis solares disponíveis"

## 🚨 PROFUNDIDADE E EXTENSÃO — REGRA DE CONTEÚDO

> Páginas com pouco conteúdo não ranqueiam e não convencem.
> Cada página de serviço deve ser completa, extensa e impressionante.

### Mínimos obrigatórios:
✔ Mínimo **6 seções** de conteúdo por página de serviço (excluindo hero e footer)
✔ Mínimo **800 palavras** de conteúdo total
✔ Cada seção deve ter conteúdo real — nunca seção vazia ou com 1 frase só
✔ Copy extensa com profundidade técnica — o cliente precisa sentir autoridade

### Após pesquisa Google — AÇÃO OBRIGATÓRIA:
✔ Se a pesquisa revelou tópicos que os concorrentes cobrem e a página NÃO → **CRIAR seção nova**
✔ Cada seção nova deve ter: H2 técnico + mín 3 parágrafos + componente DaisyUI
✔ NUNCA anotar "poderia adicionar X" sem criar a seção de fato
✔ Se seções duplicadas existem (ex: 2x mesmo tema) → MESCLAR em 1 seção
❌ PROIBIDO deixar tópico faltante como "pendência" ou "recomendação"

### Seções recomendadas (adaptar ao serviço):
1. Serviços/especialidades (cards com detalhes)
2. Diferenciais/por que nos escolher (benefícios concretos)
3. Como funciona/processo (etapas detalhadas)
4. O que está incluído (checklist completo)
5. Projetos/portfólio (cases ou números)
6. Depoimentos/prova social
7. FAQ (perguntas frequentes)
8. CTA final com formulário ou WhatsApp

---

## COMPONENTES DAISYUI — OBRIGATÓRIO USAR

> DaisyUI existe para isso — criar seções visualmente ricas e variadas.
> Nunca criar uma página "crua" usando só divs e grids manuais.

### Componentes recomendados por tipo de seção:

| Seção | Componente DaisyUI |
|-------|-------------------|
| Diferenciais | `card` com `card-body`, ícones, hover effects |
| Processo/etapas | `steps` ou `timeline` |
| FAQ | `<details>/<summary>` nativo (NUNCA DaisyUI collapse) |
| Estatísticas | `stats` com números reais |
| Depoimentos | `card` com avatar e quote |
| Checklist incluído | `list` com checkmarks |
| Alerta/destaque | `alert` com ícone |
| Tabs de conteúdo | `tabs` para organizar info |

✔ SEMPRE usar componentes DaisyUI para dar variedade visual
✔ SEMPRE combinar texto + componentes — nunca só texto corrido
✔ Cada seção deve ter um recurso visual que a diferencie das outras
✔ A página precisa impressionar e parecer premium

❌ NUNCA fazer página com só texto e títulos
❌ NUNCA repetir o mesmo layout em todas as seções (monotonia)
❌ NUNCA entregar página com menos de 6 seções de conteúdo
