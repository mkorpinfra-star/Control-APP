📋 PROCEDIMENTO DE DEPLOY:

1️⃣ UPLOAD BACKEND (12 arquivos PHP + 1 helper + 1 cron)
2️⃣ EXECUTAR SQL 1: backend/sql/create_notificacoes.sql
3️⃣ EXECUTAR SQL 2: backend/sql/notificacoes_auto_cleanup_sem_super.sql
   ⚠️ Se der erro "Acesso negado SUPER", ignorar a parte do EVENT
   ✅ A Stored Procedure vai ser criada normalmente
4️⃣ CONFIGURAR CRON JOB (no cPanel ou servidor):
   Adicionar no crontab:
   ```
   0 3 * * * /usr/bin/php /caminho/completo/backend/cron/cleanup_notificacoes.php
   ```
   Ou no cPanel: Adicionar CRON Job diário às 3h
5️⃣ BUILD FRONTEND:
   ```bash
   npm run build
   ```
6️⃣ UPLOAD FRONTEND (dist/ completa)
7️⃣ TESTAR FLUXO COMPLETO (abaixo)

📋 CONFIGURAR CRON JOB NO CPANEL:

1. Acessar cPanel → Cron Jobs
2. Na seção "Add New Cron Job"
3. Configurar:
   - Minuto: 0
   - Hora: 3
   - Dia: *
   - Mês: *
   - Dia da Semana: *
   - Comando:
     /usr/bin/php /home/seuusuario/public_html/backend/cron/cleanup_notificacoes.php

4. Salvar

⚠️ IMPORTANTE: Substituir "/home/seuusuario/public_html/" pelo caminho real do servidor!

Para descobrir o caminho real, criar arquivo PHP:
```php
<?php echo __DIR__; ?>
```
